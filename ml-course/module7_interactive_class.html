Opening Hook

Your dataset has 500 features. Training takes 3 hours. Can we keep the signal but lose the noise?

Eigenvalue Interpretation

Eigenvalues: Who's Pulling the Weight?

Each principal component has an associated eigenvalue — the amount of variance it captures. Hover over each bar to see the top feature loadings for that component.

Hover bars to see feature loadings for each principal component
Rule of Thumb (Kaiser Criterion): Keep components with eigenvalue > 1.0 — they explain more variance than a single original feature. Components to the right of the red line can usually be dropped.
t-SNE

t-SNE: When Linear Isn't Enough

PCA is linear — it can't unroll a Swiss roll or separate interleaved spirals. t-distributed Stochastic Neighbor Embedding (t-SNE) preserves local neighborhood structure, making clusters visible that PCA misses. The perplexity parameter controls how many neighbors each point considers.

Perplexity (neighbor scale)15
t-SNE projection — colors represent 4 distinct clusters in the original high-dimensional space
⚠️ t-SNE Cautions: Distances between clusters are not meaningful — only within-cluster structure is reliable. Use t-SNE for visualization, not as input to downstream models. Random seed matters!
PCA vs t-SNE

Side-by-Side: What Each Method Sees

The same dataset embedded two ways. PCA preserves global geometry; t-SNE reveals local cluster structure. Neither is "better" — they answer different questions.

PCA

✅ Global structure intact
❌ Overlapping clusters

t-SNE

✅ Clear cluster separation

When to use each:

Criterion PCA t-SNE
Interpretability ✅ High ⚠️ Low
Captures non-linearity ❌ No ✅ Yes
Use as model input ✅ Yes ❌ Usually no
Deterministic ✅ Yes ❌ No
Speed on large data ✅ Fast ❌ Slow O(n²)
Autoencoders

Autoencoders: Neural Network Compression

An autoencoder is a neural network trained to reproduce its input. By forcing data through a narrow bottleneck layer, the network must learn a compact representation. Unlike PCA, the encoder can capture non-linear structure.

Animated: data flows from input → encoder → bottleneck → decoder → reconstruction. Watch the signal travel!
Encoder: Compresses input features (e.g., 784 → 256 → 64 → 10 neurons). Each layer learns increasingly abstract representations.
Decoder: Reconstructs the original input from the bottleneck (e.g., 10 → 64 → 256 → 784). Reconstruction error drives training.
🧠 Autoencoder vs PCA: PCA finds a linear subspace; autoencoders learn non-linear manifolds. Autoencoders are more powerful but require more data and tuning. A linear autoencoder with no activation functions is mathematically equivalent to PCA!

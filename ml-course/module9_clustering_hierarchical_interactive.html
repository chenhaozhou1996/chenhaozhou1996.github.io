1. Agglomerative Clustering: Bottom-Up

Start with individuals, merge into groups

The Core Idea

Agglomerative (bottom-up) hierarchical clustering is the most common variant. It starts with each data point as its own cluster, then repeatedly merges the two closest clusters until everything is merged into one.

Algorithm:
  1. Initialize: n clusters, one per data point
  2. Compute distance matrix (all pairwise distances)
  3. Find the two closest clusters
  4. Merge them — record the merge height (distance)
  5. Update the distance matrix
  6. Repeat steps 3-5 until 1 cluster remains
  7. Cut the resulting dendrogram at desired height

Time Complexity

  • Naïve: O(n³) — building the full distance matrix and updating it n times
  • With priority queues: O(n² log n)
  • Practical limit: ~10,000–50,000 points (use K-Means for millions)

2. Reading the Dendrogram

The tree that shows everything

What is a Dendrogram?

A dendrogram is a tree diagram that records the complete history of merges. The y-axis is distance (or dissimilarity) at which clusters merged. The higher the merge, the more different the clusters were.

How to read a dendrogram:
  • Leaf nodes at the bottom = individual data points
  • Each horizontal line = a merge event
  • Height of the horizontal line = distance at which the merge happened
  • Long vertical gap before a merge = the two groups were very different
  • Cut at any height to get a specific number of clusters
Key insight — the elbow rule for dendrograms: Look for the longest vertical line you can draw without crossing any horizontal line. The height of that line suggests the natural number of clusters — cut just below the longest gap.

3. Linkage Methods: How We Measure Distance Between Clusters

Different definitions of "closest" give very different results

Once you have more than one point in a cluster, how do you measure the distance between two clusters? The answer depends on your linkage method.

🔹 Single Linkage

Distance = minimum between any pair of points

Tends to create long, "chaining" clusters. Good for detecting elongated shapes.

🔶 Complete Linkage

Distance = maximum between any pair of points

Creates compact, roughly equal-sized clusters. Sensitive to outliers.

🔷 Average Linkage

Distance = average of all pairwise distances

Balances single and complete. More robust than both.

⭐ Ward Linkage

Minimize increase in WCSS when merging

Usually the best default. Produces balanced, compact clusters.

4. Choosing the Number of Clusters

Reading the dendrogram vs. silhouette score

Method 1: Dendrogram Visual Cut

Find the longest vertical line that you can extend horizontally without crossing any merge line. The number of lines cut at that level = number of clusters.

Rule of thumb: Draw a horizontal line across the dendrogram such that it cuts through the longest vertical segment. Count how many branches you cut through — that's your k.

Method 2: Silhouette Score

After cutting the dendrogram at different heights (giving different k values), compute the silhouette score for each. Pick the k that maximizes silhouette score.

s(i) = (b(i) - a(i)) / max(a(i), b(i))
where a(i) = avg distance to own cluster, b(i) = avg distance to nearest other cluster.
Range: -1 to +1 (higher is better; > 0.5 is good).

Method 3: Domain Knowledge

The business constraint is often the most important factor. If your marketing team can only execute 4 campaigns, then k=4. The dendrogram helps you check whether a 4-cluster cut is supported by the data.

6. K-Means vs. Hierarchical: When to Use Each

Pick the right tool for your problem

🎯 K-Means

Choose K-Means when:

  • Dataset has > 50,000 rows
  • You already know (or can estimate) k
  • Clusters are roughly spherical and similar-sized
  • You need speed and scalability
  • You need to re-run frequently (streaming data)

Classic use cases: Customer segmentation at scale, image compression, document clustering

🌳 Hierarchical

Choose Hierarchical when:

  • You don't know k and want to explore
  • The hierarchy itself is valuable (taxonomy)
  • Dataset has < 50,000 rows
  • You need to cut at multiple levels
  • You want to visualize cluster relationships

Classic use cases: Gene expression analysis, product taxonomy, organizational structure analysis

A Practical Approach: Use Both

Common workflow:
  1. Run hierarchical clustering on a representative sample (5–10K rows) to understand structure
  2. Read the dendrogram to determine the natural number of clusters
  3. Run K-Means on the full dataset with the k from step 2
  4. Compare K-Means centroids with hierarchical cluster profiles — they should roughly align

📝 Module 9 — Hierarchical Clustering Summary

  • Agglomerative clustering starts with n individual clusters and merges iteratively
  • The dendrogram captures the full merge history — cut at any height for any k
  • Linkage methods define cluster distance: Ward is the best default for most business applications
  • Find k by looking for the longest vertical gap in the dendrogram
  • Practical limit: ~50K rows; use K-Means on a sample, hierarchical for structure discovery
  • Use both together: hierarchical to find k, K-Means to scale up
